# 技术预研流程

技术预研的目标不是“收集信息”，而是降低主线计划中的关键不确定性。

## 必须触发预研的场景

- 核心依赖选型
- 架构边界调整
- 成本模型重大变化
- 检索或存储方案变化
- 调度与触发模型变化
- 安全策略或沙箱模型变化
- 10x / 100x 扩容关键路径变化

## 预研必须回答的问题

1. 当前问题是什么？
2. 候选方案有哪些？
3. 对质量、延迟、成本的影响分别是什么？
4. 对 10x / 100x 扩容的影响是什么？
5. 对评测、回归和运维的影响是什么？
6. 最终建议落在哪条主线计划里？

## 输出要求

- 预研文档必须进入 `docs/research/`
- 预研结论必须回写到相关 `.trae/specs/<主线>/`
- 重大预研结论必须更新 `docs/architecture/*` 或 `docs/specs/*`
