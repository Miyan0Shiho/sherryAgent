# MVP-1 开发复盘报告

## 1. 开发过程时间线

### 项目初始化阶段
- **项目结构搭建**：pre-phoenix 阶段曾包含实现与测试目录；当前已进入 docs-only（Egg），实现资产已删除，仅保留文档契约与计划
- **核心架构设计**：采用六层融合架构，包括交互层、编排层、执行层、自主运行层、记忆层和基础设施层
- **技术栈选择**：Python 3.12+、asyncio、Textual、LLM 集成等

### 核心功能开发阶段
1. **Agent Loop 实现**：完成了核心执行循环，支持消息输入 → LLM 推理 → 工具执行 → 结果反馈的完整流程
2. **工具系统开发**：实现了文件读写、Shell 执行、HTTP 请求等基础工具
3. **权限系统实现**：建立了三层权限保护，包括工具声明式权限、全局安全规则和沙箱隔离
4. **LLM 集成**：支持 Anthropic、OpenAI 和 Ollama 本地模型
5. **CLI 界面**：实现了基于 Textual 的 TUI 界面

### 测试与验证阶段
1. **测试套件构建**：创建了完整的测试套件，包括单元测试、集成测试和端到端测试
2. **测试覆盖分析**：配置 pytest-cov 生成覆盖率报告，确保核心逻辑覆盖率 > 80%
3. **问题修复**：修复了 Agent Loop 中的 token 预算检查逻辑和测试标记配置
4. **文档完善**：创建了测试运行指南和相关文档

## 2. 错误分析与分类

### 技术错误
1. **Token 预算检查逻辑错误**
   - **现象**：Agent Loop 中的 token 预算检查逻辑有问题
   - **原因**：检查时机不正确，没有在添加 token 使用后立即检查
   - **影响**：可能导致 token 消耗超过预算
   - **修复**：调整检查时机，在添加 token 使用后立即检查

2. **测试标记配置缺失**
   - **现象**：pytest 标记未在配置中注册，导致警告
   - **原因**：pyproject.toml 中缺少标记配置
   - **影响**：测试运行时产生警告，影响测试体验
   - **修复**：在 pyproject.toml 中添加标记配置

3. **事件循环关闭异常**
   - **现象**：测试结束时出现事件循环关闭异常
   - **原因**：异步测试的清理逻辑问题
   - **影响**：测试输出中出现异常信息，但不影响测试结果
   - **修复**：优化异步测试的清理逻辑

4. **LLM 客户端覆盖率低**
   - **现象**：LLM 客户端模块覆盖率仅为 33.33%
   - **原因**：云端 LLM 客户端未完全测试
   - **影响**：可能存在未测试的代码路径
   - **修复**：补充 LLM 客户端的测试用例

### 流程错误
1. **测试脚本分散**
   - **现象**：测试脚本分散在根目录和测试目录中
   - **原因**：开发过程中临时创建的测试脚本未及时整合
   - **影响**：测试管理混乱，难以维护
   - **修复**：将临时测试脚本整合到标准测试目录

2. **文档更新不及时**
   - **现象**：部分文档未及时更新以反映实际实现
   - **原因**：开发过程中文档更新滞后
   - **影响**：文档与代码不一致，影响后续开发
   - **修复**：建立文档更新机制，确保文档与代码同步

### 沟通错误
1. **需求理解偏差**
   - **现象**：部分功能实现与预期需求有偏差
   - **原因**：需求沟通不够充分
   - **影响**：需要额外的修改和调整
   - **修复**：加强需求沟通，确保理解一致

2. **测试环境配置不明确**
   - **现象**：测试环境配置要求不明确
   - **原因**：环境配置文档不完善
   - **影响**：测试运行困难，特别是 Ollama 本地模型测试
   - **修复**：创建详细的测试环境配置指南

## 3. 成功经验总结

### 技术实践
1. **模块化架构设计**
   - **做法**：采用清晰的模块化设计，将不同功能分离到独立模块
   - **效果**：代码结构清晰，易于维护和扩展
   - **可复用性**：适用于后续 MVP 开发

2. **完整的测试覆盖**
   - **做法**：建立了多层次的测试套件，包括单元测试、集成测试和端到端测试
   - **效果**：确保代码质量，减少回归问题
   - **可复用性**：测试框架可用于后续功能的测试

3. **多 LLM 模型支持**
   - **做法**：实现了对 Anthropic、OpenAI 和 Ollama 本地模型的支持
   - **效果**：提高了系统的灵活性和可扩展性
   - **可复用性**：模型集成架构可用于支持更多 LLM 模型

4. **异步架构**
   - **做法**：使用 asyncio 实现异步操作
   - **效果**：提高系统性能，支持并发操作
   - **可复用性**：异步模式可用于后续需要并发处理的功能

### 流程实践
1. **测试驱动开发**
   - **做法**：先编写测试，再实现功能
   - **效果**：确保功能符合预期，减少 bug
   - **可复用性**：适用于后续功能开发

2. **持续集成**
   - **做法**：配置了 pytest-cov 进行覆盖率分析
   - **效果**：及时发现测试覆盖不足的问题
   - **可复用性**：可用于后续 CI/CD 流程

3. **文档先行**
   - **做法**：先创建规范文档，再进行实现
   - **效果**：确保开发方向明确，减少返工
   - **可复用性**：文档模板可用于后续 MVP 开发

### 团队协作
1. **清晰的任务分解**
   - **做法**：将复杂任务分解为小的、可管理的子任务
   - **效果**：提高开发效率，减少开发风险
   - **可复用性**：任务分解方法可用于后续项目

2. **持续沟通**
