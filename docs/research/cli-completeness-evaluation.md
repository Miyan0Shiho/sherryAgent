# CLI 功能完整性评估

## 背景

SherryAgent 的 CLI 功能目前仅通过集成测试和代码测试验证，缺乏真实使用场景的测试。本文档评估 CLI 的功能完整性，识别缺失的功能和改进方向。

## 评估范围

- 命令行参数解析（能力锚点：[IL-CLI-CommandRouter](../legacy/implementation-snapshot.md#il-cli-commandrouter)）
- TUI 界面功能（能力锚点：[IL-CLI-TUI](../legacy/implementation-snapshot.md#il-cli-tui)）
- 交互体验
- 错误处理与提示
- 测试覆盖度

## 功能评估矩阵

### 1. 命令行参数解析

| 功能 | 状态 | 完整度 | 问题描述 |
|------|------|--------|----------|
| 基础命令结构 | ✅ 已完成 | 100% | 使用 Click 框架，支持 `run`、`status`、`plugin` 命令 |
| 版本信息 | ✅ 已完成 | 100% | `--version` 选项正常工作 |
| 模型选择 | ✅ 已完成 | 80% | 支持 `--model` 参数，但缺少模型列表查询 |
| 调试模式 | ✅ 已完成 | 70% | `--debug` 参数存在，但调试信息不够详细 |
| 后台运行 | ⚠️ 部分完成 | 40% | `--daemon` 仅支持 Unix，缺少进程管理（启动/停止/重启） |
| 配置文件 | ❌ 缺失 | 0% | 无配置文件支持，所有参数需命令行指定 |
| 环境变量 | ⚠️ 部分完成 | 30% | 仅 API Key 通过环境变量配置，其他选项不支持 |
| 日志级别 | ❌ 缺失 | 0% | 无 `--log-level` 选项，无法控制日志输出 |
| 帮助文档 | ⚠️ 部分完成 | 50% | 命令有基本说明，但缺少示例和详细参数说明 |
| 参数验证 | ⚠️ 部分完成 | 40% | 缺少模型名称验证、路径验证等 |

**关键问题：**

1. **跨平台兼容性差**：`--daemon` 使用 `os.fork()`，仅支持 Unix 系统
2. **配置管理缺失**：无法保存用户偏好，每次启动需重新指定参数
3. **进程管理不完整**：缺少 `stop`、`restart`、`logs` 等后台进程管理命令

## 改进优先级排序

### P0 - 阻塞性问题（必须修复）

| 优先级 | 问题 | 影响 | 建议方案 |
|--------|------|------|----------|
| P0-1 | 后台模式跨平台兼容性 | Windows 用户无法使用后台模式 | 使用 `python-daemon` 或 `systemd` 服务替代 `os.fork()` |
| P0-2 | 缺少配置文件支持 | 每次启动需重新指定参数 | 支持 TOML/YAML 配置文件，优先级：命令行 > 配置文件 > 默认值 |
| P0-3 | 错误信息不友好 | 用户无法理解和解决问题 | 实现错误代码体系，提供友好提示和解决方案 |
| P0-4 | 缺少进程管理命令 | 后台进程难以管理 | 添加 `stop`、`restart`、`logs` 命令 |

### P1 - 体验优化（强烈建议）

| 优先级 | 问题 | 影响 | 建议方案 |
|--------|------|------|----------|
| P1-1 | 缺少进度反馈 | 用户无法判断任务状态 | 添加进度条、百分比、预估时间 |
| P1-2 | 缺少命令历史 | 重复输入成本高 | 使用 `prompt-toolkit` 或 Textual 内置历史功能 |
| P1-3 | 状态栏信息不足 | 无法监控资源消耗 | 显示 Token 消耗、响应时间、队列长度 |
| P1-4 | 缺少多行输入 | 无法输入复杂任务 | 支持多行输入模式（如 `Ctrl+Enter` 提交） |
| P1-5 | 缺少输出搜索 | 长对话难以定位信息 | 添加搜索/过滤功能（`/` 快捷键） |

### P2 - 功能增强（建议添加）

| 优先级 | 问题 | 影响 | 建议方案 |
|--------|------|------|----------|
| P2-1 | 缺少主题配置 | 无法个性化界面 | 支持主题文件，提供预设主题 |
| P2-2 | 缺少历史会话管理 | 无法回溯历史对话 | 实现会话持久化和历史浏览 |
| P2-3 | 缺少输出导出 | 无法保存对话记录 | 支持导出为 Markdown、JSON、TXT |
| P2-4 | 缺少自动补全 | 输入效率低 | 实现命令和参数的自动补全 |
| P2-5 | 缺少交互式教程 | 新用户上手困难 | 首次启动时提供引导教程 |

### P3 - 锦上添花（可选）

| 优先级 | 问题 | 影响 | 建议方案 |
|--------|------|------|----------|
| P3-1 | 缺少多会话支持 | 无法同时处理多个任务 | 实现会话标签页或窗口 |
| P3-2 | 缺少可访问性支持 | 部分用户无法使用 | 添加屏幕阅读器支持、高对比度模式 |
| P3-3 | 缺少插件扩展点 | CLI 功能无法扩展 | 支持插件注册自定义命令和 UI 组件 |
| P3-4 | 缺少快捷键自定义 | 无法适应个人习惯 | 支持快捷键配置文件 |
| P3-5 | 缺少性能监控 | 无法发现性能瓶颈 | 添加性能指标收集和展示 |

## 结论

SherryAgent 的 CLI 功能已具备基本可用性，但在用户体验、错误处理、测试覆盖等方面存在明显短板。主要问题集中在：

1. **跨平台兼容性**：后台模式仅支持 Unix 系统
2. **配置管理**：缺少配置文件和持久化机制
3. **交互体验**：缺少进度反馈、历史记录、多行输入等基础功能
4. **错误处理**：错误信息不友好，缺少恢复机制
5. **测试覆盖**：缺少真实用户场景测试和兼容性测试

建议按照优先级顺序逐步改进，优先解决 P0 阻塞性问题，再优化用户体验，最后添加增强功能。同时需要建立持续的用户反馈机制，确保改进方向符合实际需求。
