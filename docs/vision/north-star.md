# 北极星：SherryAgent 的融合愿景与成功定义

本文件是 SherryAgent 的“唯一北极星文档”。任何新增 Story、架构设计、规范与开发任务，都必须能回指这里的目标与成功定义。

## 1. 三个结论（必须成立）

### 1) Claude Code 的优势与盲区

- **优势**：CLI 驱动的严谨工程实践；强多 Agent 编排；强安全控制；强开发规范；适合“做对”。
- **盲区**：不擅长长时间自主、条件触发、重复性工作（尤其是上线后的运维/巡检/报告）。

### 2) OpenClaw 的优势与盲区

- **优势**：个人助理与自主操作；可长跑；条件满足后自启；擅长重复劳动与上线后运维、调研、报告。
- **盲区**：安全风险更高；工程规范与严谨开发流程薄弱；难以“严格、标准地开发一个项目”。

### 3) SherryAgent 的融合命题

SherryAgent 通过 **六层能力 + 场景编排（Story 验收套件）+ 权限纵深 + 规范驱动开发（以 `.trae/specs` 执行）**，把“严谨工程能力”和“自主重复劳动能力”拼成一个系统：

- 当模型能力不足时，用 **任务分解、输出约束、工具化执行、记忆与预算、审计与确认** 补齐。
- 目标不是“依赖更强模型”，而是让“具备基础能力的小模型”在复杂任务上被系统放大。

## 2. 我们如何证明成功

### 2.1 成功定义（系统工程胜过单纯换模型）

使用同一套任务集，对比两种模式在同一基础模型（优先 `qwen3:0.6b`）下的结果差异：

1. **裸奔模式**：无结构化流程、无严格权限、无多 agent 编排、无输出契约（或最少约束）。
2. **SherryAgent 加持模式**：按 Story 的演示脚本与输出契约执行，启用权限管道、编排、记忆与预算策略。

如果 SherryAgent 显著提升“完成率、可复现度、安全性与工程质量”，则融合命题成立。

### 2.2 指标（必须可量化/可审计）

- **任务完成率**：达到目标且满足输出契约的比例。
- **返工率**：需要人类二次纠正/补写/重跑的次数（或工时）。
- **危险操作拦截率**：高风险/严重风险操作被正确拒绝或要求确认的比例。
- **输出可复现度**：在相同输入与预算策略下，输出结构与关键结论可复现（允许内容细节波动）。
- **预算上限**：token/成本/时间上限明确（例如每个 Story 有默认预算与硬停止策略）。

> 注：指标的“采集方式与数据格式”在各 Story 文档的 Observability 部分定义，确保可观察与可回放。

## 3. 失败边界（必须写出来）

以下情况不承诺由流程“放大解决”，必须升级模型或引入人类介入：

- 需要深度领域知识且无法通过工具检索补齐的信息缺口
- 高风险动作需要强判断与上下文把握（系统可拦截/确认，但不能自动替你承担责任）
- 任务本身不具备可验证输出契约（无法定义“对/错”或无法观测）

## 4. 这份北极星如何约束后续开发

- 任何新增能力必须归入某个 Story（或提出新 Story），并补齐：演示脚本、输出契约、风险与权限策略、失败降级。
- 任何与安全相关的能力，必须在权限管道中落点（规则/审计/确认/沙箱），否则视为“不合格功能”。
- 任何“为了跑通而省略规范”的实现，必须登记为债务，并给出偿还 Gate（见 Distill 流程与 `.trae/specs`）。
