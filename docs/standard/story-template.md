# Story 文档模板（面试故事/场景包）

Story 是 SherryAgent 的“场景包”。它必须是一个可演示的闭环，并能映射到六层能力与权限策略。

## 1. 概述

- **Goal**：一句话说明这个 Story 要解决什么问题、为谁解决。
- **Non-Goals**：明确不做什么，避免需求漂移。
- **用户价值**：面试叙事中你要强调的价值点（工程性、安全性、可落地性等）。

## 2. 演示脚本（必须可执行）

### 2.1 手动交互演示（CLI/TUI）

- 输入是什么
- 关键中间输出是什么
- 最终输出是什么（必须满足 Output Contract）

### 2.2 条件触发/后台演示（Heartbeat/Cron/事件）

- 触发条件是什么（时间表达式/阈值/事件）
- 运行模式是什么（前台/后台/混合）
- 权限与确认点在哪里

## 3. Output Contract（输出契约）

定义“结果长什么样”，确保可复现与可审计：

- 报告格式（推荐 Markdown/JSON）
- 必须字段（例如：task_id、inputs、assumptions、decisions、actions、evidence、risks、next_steps）
- 可观察状态字段（例如：phase、progress、budget、risk_level、blocked_reason）

## 4. Observability（可观察性与回放）

- 需要记录哪些审计事件（工具调用、确认、拒绝、失败、预算耗尽）
- 需要哪些指标采集（完成率、返工率、危险操作拦截率、预算）
- 如何回放（输入 + 状态 + 审计日志即可复盘）

## 5. 风险分级与权限策略（必须写死）

按风险等级列出处理方式：

- **LOW**：自动放行 + 审计
- **MEDIUM**：自动放行 + 审计 + 通知
- **HIGH**：暂停等待确认 / 只读降级
- **CRITICAL**：直接拒绝 + 告警

并写清：
- 哪些工具/操作属于哪一类
- 哪些操作永远禁止（安全红线）

## 6. 失败模式与降级

至少覆盖：

- 模型能力不足（输出不合格/跑偏）
- 工具失败（网络/权限/路径/超时）
- 预算耗尽（token/time/cost）
- 权限拒绝（需人工确认）

并定义降级策略：
- 重试策略 / 拆分更小子任务 / 转人工 / 升级模型

## 7. 六层映射（必须写）

用一段简短说明把该 Story 映射到六层架构的关键落点：
- 交互层：输入/输出通道
- 编排层：如何分解、如何并行、如何汇总
- 执行层：Agent Loop 与工具调用策略
- 自主运行层：触发与调度方式
- 记忆层：短期/长期/桥接的使用策略
- 基础设施层：权限、沙箱、MCP/Skill、审计
