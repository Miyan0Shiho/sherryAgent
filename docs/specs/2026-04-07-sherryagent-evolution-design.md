# 系统能力总览（对齐说明）

本文档保留原 `sherryagent-evolution-design` 文件名，仅作为过渡说明使用。

## 当前口径

SherryAgent 不再以“学习项目 / demo 进化设计”为主叙事，而以 **公司级 Hybrid Platform 蓝图** 为主叙事。

当前有效入口：

- 产品定位：`docs/vision/product-charter.md`
- 系统蓝图：`docs/architecture/system-blueprint.md`
- 模块边界：`docs/architecture/module-map.md`
- 核心链路：`docs/architecture/core-operational-loops.md`
- 数据契约：`docs/specs/core-data-contracts.md`
- 实现总计划：`docs/plans/implementation-program.md`

## 已废弃的旧叙事

以下表述不再作为当前主规划口径：

- “毕业项目 / 简历项目 / 学习平台” 作为主要价值定义
- 场景层直接替代平台模块边界
- 先按独立功能 demo 分模块实现，再补治理与运营

## 保留价值

本文件的历史价值仅在于说明：SherryAgent 曾从“场景驱动、实现导向”的思路出发，现已收敛为“平台能力域 + 生命周期治理”的思路。
